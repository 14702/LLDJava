InventoryProvider — abstraction for any inventory source. Supports reserve (provisional hold), confirmReservation (reserve→block), releaseReserved (undo reserve), releaseBlocked (undo block on cancel), and fulfill (deduct stock permanently).
InternalInventoryProvider — Flipkart's own inventory. Uses ConcurrentHashMap for storage and per-item ReentrantLock for safe concurrent access. Supports addItem to stock new products or increase quantity.
ExternalInventoryProvider — simulates an external seller's API. Same InventoryProvider contract but inventory is pre-loaded (not managed by Flipkart). Cannot add items via addItemToInventory.

Enums model
OrderState: CREATED, CONFIRMED,
  CANCELLED, FULFILLED
SellerType: INTERNAL, EXTERNAL
Order — represents a customer purchase. Holds all items (same seller), address, calculated total, and tracks state transitions. createdAt used for auto-cancel timeout.
OrderItem — a line item: which product and how many units.
InventoryItem — tracks stock levels with three buckets: total, reserved (provisional), blocked (confirmed). Available = total − reserved − blocked. On fulfill, total and blocked both decrease.
OrderState — CREATED (reserved) → CONFIRMED (blocked) → FULFILLED (deducted). CANCELLED can happen from CREATED or CONFIRMED.
SellerType — INTERNAL (Flipkart-managed) or EXTERNAL (third-party seller API).
